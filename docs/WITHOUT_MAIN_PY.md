# Работа плагина без main.py

Всё необходимое реализовано **в рамках самого плагина**. main.py — только образец; интеграция с ним **опциональна**.

## Что работает без main.py

1. **HD-генератор LTC из Ltub**  
   В плагине есть класс `WC_Exchange_Quote_LTC_HD_Deriver`: вывод LTC-адреса по расширенному публичному ключу (Ltub) и индексу на **чистом PHP** (BIP32 + secp256k1).  
   Требуется расширение **bcmath** или **gmp**. В настройках включаете «HD Wallet», вводите Ltub — адреса генерируются локально, без запросов к API.

2. **Проверка оплаты**  
   Cron раз в 5 минут опрашивает **Chain.so** или **BlockCypher** по адресу заказа и при поступлении нужной суммы переводит заказ в «Оплачен» и списывает товар. Внешний API не нужен.

3. **Переадресация на оплату**  
   После «Оформить заказ» клиент перенаправляется на fluidmoney.xyz или на ваш URL страницы оплаты с подставленными суммой и LTC-адресом. Всё формируется в плагине.

## Где может участвовать main.py (по желанию)

- **Котировки** — если в настройках указан «URL API котировок» (ваш сервер с main.py или другой сервис), плагин запрашивает курс и показывает сумму в LTC на checkout. Если URL не указан, котировка при создании заказа не запрашивается (адрес и редирект всё равно работают).
- **HD через API** — если по какой-то причине локальный вывод в плагине недоступен (нет bcmath/gmp или ошибка), можно оставить включённый «URL API котировок» и добавить у себя эндпоинт вида `GET /api/v1/hd/derive?xpub=...&index=N`. Тогда плагин подставит адрес из этого API. По умолчанию плагин **сначала** пробует локальный вывод, потом API.

## Итог

Плагин **не зависит** от main.py. Локально в плагине работают:

- вывод LTC из Ltub (PHP),
- проверка оплаты (Chain.so/BlockCypher),
- обновление статуса WooCommerce.

Проблем с интеграцией не будет: при отказе от main.py достаточно не указывать URL API котировок (или указать свой) и пользоваться HD + проверкой оплаты в плагине.
