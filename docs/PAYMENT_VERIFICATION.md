# Как проверяется оплата LTC (CryptoWoo и наш плагин)

## Как CryptoWoo проверяет, что на адрес пришла оплата

1. **Блокчейн через API**  
   CryptoWoo не подключается к ноде сам. Он использует **внешние API блок-эксплореров**, которые умеют отвечать на вопрос: «Есть ли на адресе X входящие транзакции / баланс?»  
   Для LTC обычно используются, например:
   - **SoChain**
   - **BlockCypher**
   - **Litecore.io**

2. **Периодическая проверка (cron)**  
   В настройках CryptoWoo рекомендуется **отключить встроенный WordPress cron** и настроить **внешний cron** (например, раз в 1–5 минут), который вызывает задачу проверки платежей.  
   По расписанию плагин:
   - берёт заказы в статусе «ожидает оплаты» (по своему способу оплаты),
   - для каждого заказа знает адрес и ожидаемую сумму LTC,
   - запрашивает у выбранного блок-эксплорера данные по этому адресу (входящие транзакции/баланс),
   - если видит поступление нужной суммы (с учётом настроек подтверждений и zero-conf) — помечает заказ как оплаченный.

3. **Откуда берётся адрес и сумма**  
   При создании заказа через **свой** способ оплаты CryptoWoo (например «Litecoin») плагин сам:
   - генерирует/назначает LTC-адрес (из списка или HD Wallet),
   - сохраняет его в мету заказа,
   - сохраняет ожидаемую сумму LTC.  
   При проверке он читает этот же адрес и сумму из меты и опрашивает API по этому адресу.

Итого: **проверка = периодический опрос блок-эксплорера по адресу из заказа и сравнение поступившей суммы с ожидаемой.**

## Заказы нашего шлюза (Exchange Quote) и CryptoWoo

- Заказ создаётся нашим шлюзом («Оплата картой (курс Revolut)»), способ оплаты — **Exchange Quote**, а не «Litecoin» от CryptoWoo.
- Адрес LTC мы **берём из CryptoWoo** (мета заказа или HD Wallet), редирект на оплату ведёт на этот же адрес — то есть клиент платит именно на тот адрес, который выдал CryptoWoo.
- Но **стандартная проверка CryptoWoo** завязана на заказы со **своим** способом оплаты (например «Litecoin»). Заказы с методом `exchange_quote` по умолчанию в список «проверяемых» могут не попадать.

Поэтому возможны два подхода.

### Вариант A: CryptoWoo проверяет и наши заказы

Если в вашей версии CryptoWoo есть возможность «проверять любой заказ, у которого в мете есть адрес LTC и сумма» (или список типов заказов для проверки), можно включить туда заказы с методом `exchange_quote`. Тогда проверка остаётся той же: cron → API блок-эксплорера по адресу из меты → обновление статуса при поступлении суммы.

### Вариант B: Встроенная проверка плагина (реализовано)

В плагине реализована **своя проверка** по той же схеме:

- В настройках включается опция **«Проверка оплаты LTC»**.
- По cron **каждые 5 минут** запускается проверка:
  - выбираются заказы: способ оплаты = `exchange_quote`, статус = pending, в мете есть LTC-адрес и ожидаемая сумма LTC;
  - для каждого заказа запрашивается баланс по адресу через **Chain.so** или **BlockCypher**;
  - если полученная сумма (confirmed или + unconfirmed при включённом 0-conf) ≥ ожидаемой — заказ переводится в **«Оплачен»** (`payment_complete`), списывается товар, добавляется заметка к заказу.

То есть **логика проверки та же, что у CryptoWoo**: «получил ли данный адрес LTC в нужной сумме» — через запрос к блок-эксплореру. После успешной проверки WooCommerce обновляется автоматически.

## Кратко

- **CryptoWoo проверяет получение LTC так:** по расписанию (cron) запрашивает у блок-эксплорера (SoChain, BlockCypher, Litecore и т.д.) данные по **адресу из заказа** и сверяет поступившую сумму с ожидаемой.
- Адрес LTC для нашего шлюза мы как раз берём из CryptoWoo (в т.ч. HD Wallet), поэтому оплата идёт на тот же адрес.
- Чтобы именно **наш** заказ считался оплаченным, нужно либо настроить CryptoWoo на проверку заказов с методом `exchange_quote`, либо реализовать в нашем плагине такую же проверку (cron + API по адресу из меты).
