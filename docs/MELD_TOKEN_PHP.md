# Получение токена Meld в PHP (без Playwright)

## Почему не «как TokenManager + Playwright» в плагине

**Playwright** существует только для **Node.js** и **Python**. В PHP нет библиотеки, которая запускает настоящий браузер и перехватывает сетевые запросы. Поэтому точную копию TokenManager из `main.py` (Playwright + fluidmoney.xyz) **в самом плагине на PHP реализовать нельзя**.

## Варианты

### 1. Токен получает ваш API (рекомендуется)

Сейчас плагин вызывает **ваш Exchange Quote API** (`api_base_url`). Токен Meld получает и обновляет только **main.py** (TokenManager + Playwright). PHP токен не хранит и не обновляет.

- Плюсы: один раз настроил API — токен всегда актуален, не нужен Python на хостинге магазина.
- Минусы: магазин должен быть способен достучаться до вашего API (сеть, firewall).

### 2. Токен обновляет скрипт на сервере, плагин читает из файла

Если на **том же сервере**, где работает WordPress, есть **Python 3 + Playwright + Chromium**, можно:

1. Запускать по крону (например раз в 20 минут) скрипт, который получает токен так же, как TokenManager в main.py, и **записывает его в файл** (например `/tmp/meld_session_token.txt`).
2. В плагине включить опцию «Использовать токен из файла» и указать путь к этому файлу. Плагин будет читать токен из файла и вызывать Meld API напрямую (без вашего API).

Плюсы: котировки идут напрямую в Meld из PHP, ваш API для котировок не обязателен.  
Минусы: на хостинге должны быть Python, Playwright и браузер; нужен cron.

Пример крона (каждые 20 минут):

```bash
*/20 * * * * /usr/bin/python3 /path/to/woo-exchange-quote-gateway/scripts/get_meld_token.py --output /tmp/meld_session_token.txt
```

Скрипт `scripts/get_meld_token.py` можно взять из репозитория exchange-quote-api или сделать минимальную копию логики TokenManager с выводом токена в файл.

### 3. PHP вызывает внешний скрипт (exec)

Теоретически плагин может по запросу вызывать `python3 get_meld_token.py` через `shell_exec()` и читать stdout. На практике:

- На shared-хостинге часто отключён `exec`/`shell_exec`.
- Запуск браузера из PHP при каждом запросе котировки даёт большую задержку (5–15 сек).
- Надёжнее запускать скрипт по cron и хранить токен в файле (вариант 2).

## Итог

| Подход                         | Токен получает      | Где хранится | Нужен Python на сервере магазина |
|--------------------------------|---------------------|--------------|----------------------------------|
| API (текущий)                  | main.py (Playwright)| В памяти API | Нет                              |
| Файл + cron-скрипт (вариант 2) | Скрипт (Playwright) | Файл         | Да                               |

«Реализовать в PHP как TokenManager + Playwright» буквально нельзя; реализуемо **использование в PHP токена, который получил внешний процесс** (API или скрипт с записью в файл).
